# Cost and Operational Readiness Checklist

## Cost Readiness
- RFC must include expected infrastructure cost.
- RFC must include third-party service cost assumptions.
- RFC must include cost drivers such as storage, compute, network, and API calls.
- RFC must define cost monitoring approach.

## Operational Readiness
- RFC must define support ownership.
- RFC must define deployment approach.
- RFC must define rollback ownership.
- RFC must define on-call impact.
- RFC must define release communication plan.

## Governance
- RFC must identify approval owners.
- RFC must list major risks and mitigations.
- RFC must identify whether architect review is required.
